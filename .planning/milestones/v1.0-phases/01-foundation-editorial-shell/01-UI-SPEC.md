# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Source note:** This phase's design system was locked by the user in an interactive session (5 prototype iterations rejected, 1 approved) *before* this contract was written. This document formalizes those locked decisions — it does not introduce new design opinions except where explicitly marked "Claude's discretion (recommendation)". Canonical source: `.planning/reference/prototype-shell-and-fig01.html` (exact CSS), `.planning/phases/01-foundation-editorial-shell/01-CONTEXT.md`, `.planning/PROJECT.md`.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — no shadcn/component library. Explicit project decision (01-CONTEXT.md "Stack & structure"; STACK.md "What NOT to Use"): **no Tailwind, no UI framework/islands**. Plain CSS custom properties, hand-authored `.astro` components with scoped `<style>` blocks. |
| Preset | not applicable |
| Component library | none — every component is a bespoke `.astro` file; all colors/spacing/type read from `src/styles/tokens.css` via `var(--token)`, never a literal (ARCHITECTURE.md Anti-Pattern 2 applies to CSS too, not just the canvas module) |
| Icon library | none in Phase 1. No icon font/SVG icon set is used: status dot is a 6×6px CSS circle (`background:var(--good)`), arrows are literal `→` glyphs inside mono text (`résumé →`), Fig.01's device glyph is a hand-drawn canvas primitive (Phase 2, not this phase) |
| Font | Self-hosted subsetted `.woff2`: **serif display** + **mono**. Sans body uses the **system stack only** (no file to download) — `"Helvetica Neue","Segoe UI",Arial,system-ui,sans-serif`. See Typography section for exact faces and rationale. |

### Font faces

| Role | Face | Source | Notes |
|------|------|--------|-------|
| Serif display | **Source Serif 4**, weight 500 — Claude's discretion (recommendation); open (SIL OFL), transitional/old-style character matching the Iowan/Palatino brief, self-hostable, subsettable to a static woff2 at the single weight used | `self-hosted, public/fonts/` | Alternative if unavailable/unsuitable at scaffold time: **Literata** (also OFL, similarly warm old-style proportions). Fallback stack (locked, do not change): `"Iowan Old Style","Palatino Linotype",Palatino,Georgia,serif` |
| Mono | **Cascadia Code** — Claude's discretion (recommendation); SIL OFL, already the 2nd name in the locked fallback stack, so a self-host load failure degrades to a face the design already treats as equivalent | `self-hosted, public/fonts/` | Fallback stack (locked): `ui-monospace,"Cascadia Code","SF Mono",Consolas,monospace` |
| Sans (body) | System stack only — **no self-hosted file** | n/a | Locked stack: `"Helvetica Neue","Segoe UI",Arial,system-ui,sans-serif`. Not LCP-critical the way the hero serif is; omitting a custom download here protects the Lighthouse/CLS budget per PITFALLS.md Pitfall 4 |

Both self-hosted faces: `font-display: swap`, `<link rel="preload" as="font" type="font/woff2" crossorigin>` for the serif (above-the-fold, LCP-relevant) and mono (used in the header, first paint), plus a metrics-matched fallback (`size-adjust`/`ascent-override`/`descent-override`) tuned to each chosen face so the fallback→custom swap does not reflow the hero (PLAT-04, Pitfall 4). Subset to Latin + numerals + the specific symbols used (`·`, `→`, é in "résumé").

## Supplementary: Page Structure & Motion (context for executor, not template-required)

### Section order (single-scroll home page)
1. Header (mono identity + nav)
2. Hero (eyebrow → serif thesis → bio → 3 quiet links)
3. *[Fig. 01 slot — Phase 2 renders nothing here in Phase 1]*
4. `selected systems` — 4 rows, grid `64px 1fr auto` (year | mono name + sans description | accent metric), row hover → `--panel2` background, no links yet (Phase 3 adds case-study links)
5. `experience` — Microsoft / AWS / MathWorks / Samsung
6. `patents & publications` — 2 patents + 1 paper
7. `skills` — prose intro + grouped mono tags (no skill bars, no logo walls — explicit anti-pattern exclusion)
8. `contact` — mailto + LinkedIn (phone number excluded)
9. Footer — status line + live clock

### Motion doctrine (Phase 1 scope)
Phase 1 shell is **essentially static**. The only "live" element is the footer clock (`setInterval`, ≤30s tick). Link/button hover transitions use `border-color .15s` / `color .15s` (≤150ms, matches the project's global hover-response ceiling). `prefers-reduced-motion` has no Phase 1-specific branch to build (nothing animates besides CSS hover transitions, which are not motion-sickness-relevant) — the doctrine becomes load-bearing starting Phase 2 (Fig. 01).

### Responsive
Mobile breakpoint at `640px`: `.row` grid collapses to `52px 1fr` and hides the metric column (`.mt { display:none }`); Fig.01 canvas height drops from 380px to 300px (Phase 2). `.page` horizontal padding is fluid via `clamp(18px, 4vw, 32px)` from 360px through desktop — no other breakpoints are defined in the locked prototype; do not invent additional ones without a design update.

### Focus states (accessibility, day one)
Every link/button must show a visible focus outline (explicit requirement in 01-CONTEXT.md "Specific Ideas"). No `outline: none` without a replacement focus style. This applies even though Phase 1 has no interactive canvas — header nav, hero links, and contact links all need keyboard-visible focus.
