<objective>
Re-architect `scripts/verify-contrast.mjs` to sample REAL post-composite screenshots (the only honest way to measure through a `backdrop-filter` blur kernel), then record the pre-glass baseline that the Phase 8 glass gate is judged against. This is the phase's hard precondition (08-CONTEXT.md, GLS-03): no glass value locks until this verifier exists and the baseline is recorded. NO glass CSS is written in this plan.

Purpose: analytic per-pixel compositing (the existing `--cdp` mode) is mathematically exact for flat-alpha layers but CANNOT represent a spatial blur (a blurred pixel is a weighted neighborhood average, not a function of one pixel). Phase 7 deferred screenshot sampling to this phase precisely because there is now a blur kernel to measure.

Output: a `--cdp-screenshot` gate mode + a `--agreement-selftest` mode + a static solid fixture + a recorded pre-glass baseline table.
</objective>
