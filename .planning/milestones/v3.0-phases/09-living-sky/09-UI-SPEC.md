# Phase 9 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This is an ADDENDUM to `05-UI-SPEC.md` (night-sky scene), `05.1-UI-SPEC.md` (moon + meteor), `07-UI-SPEC.md` (real photo placement/tone), and `08-UI-SPEC.md` (glass system)** — not a replacement. Every token, placement geometry, contrast floor, and glass grammar already locked there applies unchanged unless explicitly superseded below. This document governs ONLY the four ambient systems that make the sky *live*: two drifting cloud layers, panel-change parallax, a breathing aurora, and star scintillation — how they LOOK (placement, palette, softness) and how they MOVE (speed, easing, cadence). > **Scope boundary:** this document does not re-tune glass values, the photo master, or any Phase 7/8 contrast number — those are locked. Where an ambient system risks a contrast regression, the fix is always in THIS document's placement/alpha, never in glass or scrim.
>
> **Reference viewports:** 1440×900 and 1280×800 (matching every prior phase's check-width pair). Composition geometry is stated as viewport-height/width fractions first, reference-pixel values second.

## Clouds (AMB-01)

Two pre-rendered, semi-transparent sprite layers, generated once per size (mirrors `starfield.ts`'s Layer-0 generation pattern — regenerated only on init + debounced resize, never per-tick), wraparound-blitted every frame. Procedural (layered radial gradients), tokens-only, zero image assets.

### Placement — lower sky, below the Milky Way core's visual weight

Both layers hug the horizon, well below the band's brightest region (upper-to-mid frame, right margin) and — via the column governor below — structurally attenuated to near-zero under the content column, so they pass the contrast gate by construction rather than by luck.

| Layer | y-band (fraction of viewport height) | 1440×900 (px) | 1280×800 (px) | Character |
|---|---|---|---|---|
| **Far** ("wisps") | center `0.72H`, spread `0.64H–0.80H` | 648px center, 576–720px | 576px center, 512–640px | Thin, elongated horizontal cirrus streaks |
| **Near** ("banks") | center `0.78H`, spread `0.71H–0.85H` | 702px center, 639–765px | 624px center, 568–680px | Rounder, denser cloud-bank clusters, base grazing the horizon line (`0.85H`) |

Both bands sit clear of the header (`y:0–~50px`) and the footer chrome (`y > ~0.91H`) at every reference viewport — no geometric overlap with the two glass surfaces closest to the contrast floor (worst measured today: header `6.23:1`).

### Column governor (contrast-by-construction, smoothstep — not a hard cut)

Clouds span the FULL viewport width (continuous drift needs continuous coverage), so unlike aurora's hard margin exclusion (below), they need a smooth attenuation ramp across the content-column boundary — a hard cutoff would show a visible "clean edge" artifact as a cloud drifts across it.

- Reuses the SAME `contentColumnEdges(cssWidth)` formula already mirrored in `scene.ts`/`starfield.ts` (pad `clamp(18,4vw,32)`, half `min(880,width−2·pad)/2`). At 1440×900: column `280–1160px`. At 1280×800: column `200–1080px`.
- Attenuation: alpha inside the column multiplied by **0.15×**, ramping via smoothstep over an **80px** edge zone at each column boundary (e.g. at 1440: full drop-off between `200–280px` and `1160–1240px`) — same spirit and ramp width as the photo's own build-time column vignette (`07-UI-SPEC.md` Tone Contract, `0.12×`/`80px`), applied live instead of baked.
- Result: far-layer peak alpha 0.08 → **0.012** under text; near-layer peak alpha 0.13 → **0.0195** under text — both visually negligible, well inside the existing 4.5:1 headroom (worst measured sky-over-text region today: `13.05:1`).

### Alpha ceilings (margins, full strength)

| Layer | Peak alpha (margins) | Rationale |
|---|---|---|
| Far | **0.08** (tuning range 0.05–0.10) | Barely-there haze; stars must read clearly through it |
| Near | **0.13** (tuning range 0.10–0.16, hard ceiling 0.18) | Visibly cloud-like without ever approaching the aurora's or MW's brightness |

### Drift speed (near faster — physically honest)

| Layer | Speed | Feel |
|---|---|---|
| Far | **3 CSS-px/s** | Sub-threshold — a frame-to-frame delta invisible at a glance; only apparent comparing two moments 10+ seconds apart |
| Near | **6 CSS-px/s** | Still glacial, but perceptibly faster than Far when both are visible — the depth cue |

Fixed CSS-px/s constants (not viewport-scaled), matching the firefly speed convention already in `scene.ts` (4–8px/s).

### Sprite generation — wraparound blit technique

Each layer's offscreen canvas is generated at exactly `cssWidth` wide × band-height tall, with its rightmost ~120px of blob content mirrored/repeated at the left edge during generation (standard seamless-tile technique) so the bitmap tiles losslessly left-to-right. Per frame, `scene.ts` draws the SAME tile twice, at `x = (offset mod cssWidth) − cssWidth` and `x = (offset mod cssWidth)` — full viewport coverage at any offset, zero visible seam or repeat-jump.

### Sprite softness / scale language

| Layer | Blob shape | Aspect ratio | Density | Edge softness |
|---|---|---|---|---|
| Far (wisps) | Elongated horizontal streaks | 6:1–10:1 (w:h) | 4–6 blobs across one tile width | Gradient falloff over 65–70% of blob radius |
| Near (banks) | Cluster groups of 3–4 overlapping rounder blobs | 2:1–3:1 per blob | 3–5 clusters across one tile width | Falloff over 55–65% of radius — slightly more defined than Far, still zero hard edges |

Every blob uses a **≥4-stop** radial gradient (never a bare 0%/100% stop pair) to avoid 8-bit banding on the large, slow-alpha gradients clouds require — verify with `node scripts/verify-banding.mjs --selftest` before shipping either sprite generator (context's closing battery explicitly calls out clouds/aurora as new-gradient risk).

### Color

Both layers fill with `--milkyway` (`#cfd9f2`) at the alphas above — a pale, cool, blue-white haze consistent with thin cloud scattering ambient starlight at a genuinely dark site (never a warm/city-glow tint, preserving the "zero light pollution camping" mood). `--sky-horizon` is an available secondary gradient-stop tone for a cooler outer falloff, planner's discretion, never the dominant fill.

### Mobile shedding

Far layer sheds FIRST (see Composition Doctrine's mobile ladder for exact trigger). **Visual acceptance:** with Far shed, the Near layer alone — denser, higher-alpha, larger cluster shapes — must still unambiguously read as "clouds," not an indistinct smear. Verify by screenshot at the narrowest supported width with Far layer forcibly disabled.

## Parallax (AMB-02)

### Depth model (locked, 09-CONTEXT.md) — restated with exact numbers

| Layer | Element | Shift amount | Mechanism |
|---|---|---|---|
| Foreground ("ground") | `.camper` (existing DOM div — SVG silhouette + copper glow) | **18px** (ceiling 24px) | CSS `translate3d`, compositor-only |
| Mid ("horizon") | Both cloud layers | **9px** equivalent | Canvas-internal draw-offset tween (added on top of the continuous drift offset, NOT a CSS transform) |
| Infinity | Photo `<img>` + all canvas-drawn stars/MW/moon | **0px** | Never shifts — the canvas element itself is NEVER CSS-transformed (09-CONTEXT.md's locked resolution to the canvas-transform trap) |

No separate ground/horizon silhouette exists in the DOM (the flat ground-fill rect was removed in Phase 7's Seam Contract) — `.camper` is the ONLY foreground DOM element, and is therefore the entire "ground" parallax layer.

### Direction metaphor

Advancing to the next panel (deck index increasing — moving forward/deeper into the deck) reads as the camera drifting forward-right past the scene. Real parallax: as the camera translates right, nearer objects apparently shift LEFT relative to the frame, farther objects shift less.

- **Advance (index +1):** `.camper` translates **−18px** (left); clouds' offset tween adds **−9px** (left) on top of their continuous drift.
- **Retreat (index −1):** mirrored — `.camper` translates **+18px** (right); clouds add **+9px** (right).
- Photo + stars: always `0` — they are at infinity, they never move regardless of direction.

### Duration / easing — deck.css's own family, slightly offset for depth

- **Ground (`.camper` translate3d):** `420ms cubic-bezier(0.16, 1, 0.3, 1)` — the EXACT same duration and curve as `deck.css`'s panel `transform`/`opacity` transition (line 79–81) and the Phase 7 LQIP fade, starting at `t=0`, synced with the panel swap itself.
- **Clouds (canvas offset tween):** starts **60ms** after the panel-change event fires, runs **480ms**, same `cubic-bezier(0.16, 1, 0.3, 1)` curve. The small delay + slightly longer duration is the "slightly offset" depth cue the brief calls for: the mid layer visibly settles a beat after the foreground, mimicking how smaller-amplitude motion perceptually lags larger-amplitude motion in real parallax.
- Both tweens use ease-out shape (front-loaded motion, settling gently) — never linear, matching every other motion contract in this codebase.

### Reduced motion

Both offsets apply **instantly** (`transition: none` / 0ms tween, jump directly to the new panel's target value) — "stop, don't dampen" (`PITFALLS.md` Pitfall 4), identical doctrine to the LQIP fade and camper-glow pulse. Parallax NEVER sheds on mobile regardless of the degradation ladder (09-CONTEXT.md, locked) — it is event-driven and compositor-only, effectively free.

## Aurora (AMB-03)

Noise-table-driven curtains, precomputed value-noise (zero deps), painted to an offscreen buffer throttled to every 4 frames, composited into the visible canvas every frame at a live-interpolated alpha.

### Placement — left margin only, hard-excluded from the content column

The Milky Way core owns the right margin (07-UI-SPEC placement map: core `x:0.80–0.98`); aurora is placed exclusively in the LEFT margin — the same side the moon already occupies (05.1-UI-SPEC), keeping the composition balanced (MW dominant right, aurora + moon left, camper anchors bottom-left).

Unlike clouds, aurora curtains never move horizontally (only their alpha/height "breathe"), so a **hard** margin boundary is invisible/static and safe — no smoothstep governor needed. This is a stronger, simpler contrast guarantee: aurora renders **zero pixels** inside the content column, by construction, at every frame.

| Viewport | x-range (left margin, `contentColumnEdges` + 8px cushion) | Margin width |
|---|---|---|
| 1440×900 | `x: 8–272px` | 264px |
| 1280×800 | `x: 8–192px` | 184px |

Below the existing `TWINKLE_MIN_MARGIN_WIDTH` (48px) threshold, aurora simply does not render — same graceful-empty behavior the twinkle-star margin pool already exhibits on narrow viewports (no special-case code needed, same governing constant reused).

### Vertical box

| Edge | Value | Notes |
|---|---|---|
| Base (fixed) | `y: 0.85H` (horizon line) | Curtains fade to zero exactly at the existing horizon — same line the camper/ground system already anchors to |
| Top (breathing) | oscillates `0.49H–0.55H` | Rises/falls gently in phase with the alpha breath (see below) |

1440×900: top `441–495px`, base `765px`. 1280×800: top `392–440px`, base `680px`.

The moon (fixed at `x:0.3×columnLeft`, `y:0.68H` — inside this same box) sits within the aurora curtain area. Since aurora draws AFTER the Layer-0 blit (which bakes the moon) and BEFORE clouds, it composites on top of the moon — physically correct (aurora, ~100km altitude, is vastly closer than the moon) and produces a deliberate, pleasant effect: the moon dims very slightly through the curtain at breathing peaks. This is intentional, not a defect.

### Curtain count / geometry

**3 curtains**, varying widths and heights within the margin box, edges undulating via the noise table (organic ribbon silhouette, not a hard-edged rectangle). Each curtain's top edge varies independently within the box's oscillation range so the three never breathe in perfect lockstep (avoids a mechanical, synchronized look).

### Palette

`--aurora` (`#bfe8df`, see Color section) is the sole fill. No gradient toward a second hue — a single consistent teal-white, never a rainbow (09-CONTEXT.md, locked). Vertical alpha falls off toward both the top edge (soft fade-in) and the base (blends into the horizon), using the same ≥4-stop-gradient no-banding discipline as clouds.

### Peak alpha + luminance ceiling vs. the Milky Way core

| Parameter | Value |
|---|---|
| Breathing midpoint | 0.12 |
| Breathing amplitude | ±0.05 (range **0.07–0.17**) |
| Hard ceiling | **0.20** (never exceeded regardless of tuning) |

Extend the `--moon` mode's exact comparator pattern (`scripts/verify-contrast.mjs`) with a sibling `--aurora` mode: sample the aurora's peak composited pixel within its bounding box at both reference viewports, sample the photo's Milky-Way-core peak pixel (existing sample point B, `x:0.90×cssWidth, y:0.50×cssHeight`), and assert `luminance(aurora) < luminance(mwCore)` strictly — mirroring `moonPeak < mwPeak`. Design-time estimate (not a substitute for the gate): `--aurora` at its own full-color relative luminance ≈0.74, composited at peak alpha 0.17–0.20 over a near-black background (≈0.02) yields an estimated composited luminance of **≈0.13–0.15** — comfortably below the moon's own composited luminance (≈0.41–0.45, itself already asserted below the MW core) — so this should pass with wide margin. The screenshot/luminance gate is authoritative; if it shows a shortfall, aurora dims or its box shrinks — glass and the photo grade stay locked (09-CONTEXT.md).

### Breathing cycle

**~20s period** (locked, 09-CONTEXT.md). Both alpha (0.07–0.17) and curtain-top height (`0.49H–0.55H`) oscillate together, same phase, same sine — the "inhale/exhale" reads as a single coherent swell, not two independent wobbles.

### Update cadence — the anti-stutter spec

- Brightness (alpha) is a pure sine formula evaluated fresh every rendered frame (60fps) — perfectly smooth, zero stutter risk, effectively free.
- The curtain SHAPE (noise-buffer repaint) is throttled to every **4 frames** (~66.7ms at 60fps) — the only throttled part. Max step delta: the noise-table phase advances by an increment bounded to **≤1.5px of curtain-edge x-displacement per repaint step** at the 1440 reference width. At 4-frame throttling this yields a full undulation cycle spanning 250+ repaint steps (16+ seconds) — the shape visibly ripples continuously and slowly; it never "jumps." Mobile-ladder throttle (every 8–10 frames, see Composition Doctrine) uses the SAME per-step delta ceiling, just spaced further apart in time — slower breathing, never choppier breathing.

## Scintillation (AMB-04)

Upgrades the existing ~40-star twinkle subset's single-oscillator sine wobble (`scene.ts` `seedTwinkles`/`drawFrame` Layer 2a) to a 2-oscillator sum, plus an occasional chromatic nudge on the brightest few. No count widening — same margin-confined pool, same `TWINKLE_SUBSET_FRACTION` (0.5), same ~40-star reference count.

### Amplitude — 2-oscillator sum vs. today's single oscillator

| Oscillator | Period | Amplitude | Role |
|---|---|---|---|
| Primary (existing, retuned) | 2000–4000ms (unchanged) | **0.12–0.18** (was 0.15–0.25) | The slow, existing wobble — amplitude trimmed slightly to make room for the secondary term |
| Secondary (NEW) | 400–700ms | **0.04–0.07** | Fast atmospheric flutter — the "shimmer" real stellar scintillation has, sub-second and non-sinusoidal-feeling once summed with the slow term |

`alpha = clamp01(baseAlpha + amp1·sin(ts/period1+phase1) + amp2·sin(ts/period2+phase2))`. **Combined peak envelope: ≈0.16–0.25** — statistically similar total range to today's single-oscillator 0.15–0.25, so the upgrade reads as "more textured, more alive," not "brighter" or "busier" (subtle upgrade, not disco — 09-CONTEXT.md, locked).

### Chromatic nudge

- **Which stars:** the **3 brightest** twinkle-pool entries only (hard cap — selected once per Layer-0 regeneration as the 3 largest-radius entries in the pool, deterministic per generation, never reselected per-frame — avoids a distracting "which star is doing it" flicker).
- **Color pair:** reuses `starfield.ts`'s existing `jitterStarColor` cool/warm hue pair verbatim — cool `210°` / warm `35°` — never a new hue.
- **Max saturation shift:** ±8% absolute HSL-S delta around the star's own already-jittered base saturation, modulated at the SAME period as the primary (slow) oscillator so color and brightness pulse together — mimicking real atmospheric color-fringing (chromatic dispersion), not an independent color cycle.
- **Frequency:** continuous but bounded — the ±8% swing is a small fraction of the total saturation range (jitterStarColor caps at 0.3 max), so even at its extremes the nudge stays a "nudge," never a saturated color flash.

### Photo-stars-static asymmetry

Accepted per research precedent (09-CONTEXT.md): the photo's own baked-in stars remain fully static while the drawn twinkle subset animates. This asymmetry already exists today (twinkle vs. static photo stars) and reads fine — the drawn layer is understood as "the authored layer," scintillation simply makes that authored layer read more atmospheric.

## Composition Doctrine — the "one perceptible motion" budget, operationalized

At any glance, at most one thing should read as "moving." This is enforced by keeping every CONTINUOUS ambient system's rate of change at or below a sub-perceptual-threshold budget, reserving all high-amplitude, high-velocity motion for EVENT-triggered or intentionally-rare systems.

| System | Type | Rate | Classification |
|---|---|---|---|
| Clouds (far) | Continuous | 3px/s | Sub-threshold drift — "atmosphere," not "movement" |
| Clouds (near) | Continuous | 6px/s | Sub-threshold drift |
| Aurora breathing | Continuous | ≤1.26% of its alpha range per second (0.10 total swing / 20s period, sine-shaped so peak rate is ~π/2× the average ≈ ~2% /s at the steepest point) | Glacial — imperceptible frame-to-frame, only apparent over many seconds |
| Scintillation | Continuous, high-frequency | Intensity-only (0.16–0.25 alpha range at up to 400–700ms period) — **zero spatial displacement**, confined to ≤2px-radius points | Texture, not motion — no shape moves, only brightness flickers on point-sized targets; total affected screen area ~40 points × ≤2px radius is de minimis |
| Parallax burst | Event-triggered only | 9–18px over 420–480ms (≈20–40px/s during the burst) | The ONE deliberate high-velocity motion — ~5–15× faster than any continuous system, but brief (<0.5s), user-caused (tied directly to the visitor's own panel-change action), and silent between events (zero motion the other 99%+ of the time) |
| Meteors (existing, unchanged) | Rare event | Fast, high-alpha (0.95) | The rare gift — already the sole "eye-catching" motion event by design; unaffected by this phase |

**Rule, stated as a budget:** no continuous ambient loop (clouds, aurora, scintillation) may exceed an effective visual displacement of 8px/s OR a value-change rate of 2% of its own range per second. Parallax and meteors are the only two systems permitted to exceed this, and both do so briefly and infrequently — parallax for under half a second per user action, meteors for a few seconds every few minutes. Because the continuous systems all sit well under this ceiling simultaneously, their combined presence never competes with either event system for the viewer's attention, and no two continuous systems compound into something that reads as "busy" — each is independently sub-threshold, and sub-threshold + sub-threshold is still sub-threshold.

### Mobile degradation ladder (order locked, 09-CONTEXT.md; triggers Claude's discretion)

| Tier | Trigger | Action |
|---|---|---|
| 1 | `viewport width < 640px` OR `navigator.deviceMemory <= 4` (when available) | Shed the far cloud layer entirely |
| 2 | `viewport width < 480px` OR `deviceMemory <= 2` | Throttle aurora shape repaint from every 4 frames to every **8–10 frames** (brightness sine stays per-frame, unaffected) |
| 3 | `viewport width < 390px` OR (`deviceMemory <= 2` AND tier 2 already active) | Drop the scintillation chromatic nudge (2-oscillator amplitude wobble stays; color stays neutral) |

**Parallax never sheds at any tier** (locked). `navigator.deviceMemory` is feature-detected — its absence (Safari/iOS, which never ships it) falls back to width-only thresholds, consistent with the project's existing `requestIdleCallback` fallback-as-first-class-path precedent (`starfield.ts`).

| Layer | Reduced-motion state |

## Accessibility / Floors

- **Decorative canvas, unchanged.** All new draw calls (clouds, aurora, scintillation) happen inside the existing `#nightsky-canvas`, already `aria-hidden="true"` via its host div, already `pointer-events: none`. No new ARIA surface is introduced.
- **Contrast duty — pass by construction, gate as defense-in-depth.**
  - Clouds: column-governor attenuation (×0.15, 80px smoothstep ramp) drives worst-case under-text alpha to ≤0.02 — negligible against the existing 4.5:1 floor (worst measured sky-over-text region today: `13.05:1`; the tightest surface site-wide, chrome header/footer at `6.23:1`, sits geometrically outside every cloud/aurora placement box — see Clouds Placement).
  - Aurora: hard margin exclusion — zero pixels render inside the content column at any frame, by construction, no governor needed.
  - Both are still exercised by the full closing battery (`--cdp-screenshot` gate, both viewports, all surfaces) as defense-in-depth, per 09-CONTEXT.md's Battery section. Any regression is fixed here (placement/alpha), never in glass or scrim.
- **No vestibular triggers (WCAG 2.3.3, Animation from Interactions).** Parallax is the only interaction-triggered motion this phase adds. It satisfies the SC's spirit on two independent grounds: (1) it is fully disable-able — `prefers-reduced-motion: reduce` collapses it to an instant, zero-duration jump, matching the project's standing "stop, don't dampen" doctrine applied to every other motion element in the scene; (2) even for visitors who have not set that OS preference, the effect is small (≤24px, under 2% of a 1440px viewport width), brief (≤480ms), and a translation only (no rotation, no scaling, no parallax-scrolling-induced depth-of-field effect) — well short of the large-scale, sustained, or looping motion patterns WCAG 2.3.3 guidance calls out as vestibular triggers. (WCAG 2.3.3 does not itself define a numeric px/duration threshold; this is a reasoned application of its stated purpose, not a claim of a specific normative number.)
- **Aurora/moon interaction is intentional**, not a bug — see Aurora Placement above.

**No new DOM elements.** No ground-wrapper div is needed: `.camper` (existing `NightSky.astro` div) is the sole foreground DOM element and the natural, sufficient target for the parallax `translate3d` — its `transform` property is currently unused (only `.camper-glow`'s own `opacity` animates today), so applying the parallax transition directly to `.camper` introduces zero risk of clobbering an existing transform.
