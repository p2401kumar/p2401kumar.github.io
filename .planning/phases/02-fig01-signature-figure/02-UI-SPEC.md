## Motion & Interaction Contract (supplementary â€” not template-required, but load-bearing for this phase)

### Reduced motion (FIG-05, locked doctrine from `02-CONTEXT.md` + Pitfall 3)

| State | Behavior |
|-------|----------|
| `prefers-reduced-motion: reduce` detected at init | **No rAF loop runs.** Render exactly one complete static frame: all routes drawn at full length (`prog=1`), all nodes placed at rest position (`rise=0`, `alpha=1`), no ambient beams ever spawn. |
| `send request` under reduced motion | No-op, or an instant full-opacity glow flash on the destination node with no traveling beam (Claude's discretion â€” recommend **no-op** to keep the reduced-motion path maximally simple/predictable; the figure's informative content is the topology + fault-injection story, not the send-beam flourish) |
| `inject fault` under reduced motion | **Still fully functional** â€” instantly redraws the static frame with the degraded cell's amber/dashed styling and dead-end route dashing applied, and the event log still narrates (`fault injectedâ€¦` / after 8s, `recoveredâ€¦`). | Media query change listener | Must listen for `matchMedia('(prefers-reduced-motion: reduce)').addEventListener('change', â€¦)`, not just check once at load (Pitfall 3) â€” re-branch render path live if the OS setting toggles mid-session |
