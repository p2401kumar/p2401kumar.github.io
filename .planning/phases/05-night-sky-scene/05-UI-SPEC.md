# Phase 5 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Source note (AUTO MODE — formalization of a locked vision, not open design):** the user's zero-light-pollution night scene is fully locked by `05-CONTEXT.md` (sequencing, layer split, event contract, honesty gate) and `PROJECT.md`'s v2.0 milestone description. This document translates that locked vision into exact numbers — the new sky tokens, gradient composition, star density/magnitude bands, Milky Way placement relative to the content column, silhouette/glow spec, firefly spec, constellation visual treatment (mirroring Fig. 01's beam math per `02-UI-SPEC.md`), scrim spec, and the reduced-motion static-frame composition. Where `05-CONTEXT.md` explicitly left a gap ("Claude's discretion"), this document makes the call and marks it clearly as a **recommended starting value** — several are true unknowns pending the Milky Way visual spike that opens this phase (`05-CONTEXT.md` Sequencing) and MUST be re-validated visually during that spike, not treated as final art direction. This spec extends `01-UI-SPEC.md` (shell tokens), `02-UI-SPEC.md` (Fig. 01 beam math, mirrored not imported), and `04-UI-SPEC.md` (deck chrome the scene renders behind) — it never contradicts them.

## Spacing Scale

Phase 1's baseline scale (`xs:4 / sm:8 / md:16 / lg:24 / xl:32 / 2xl:48 / 3xl:64`) applies to any ordinary DOM/CSS spacing this phase touches (there is very little — the scene canvas is a fixed full-bleed element with no padding/margin of its own). All scene-specific measurements below are canvas/composition geometry (fractions of viewport, px radii, angles) — same "exceptions table" pattern `02-UI-SPEC.md` used for Fig. 01's canvas stage.

### Scene composition geometry (locked ranges from `05-CONTEXT.md` + this phase's exact recommendations)

| Element | Value | Rationale |
|---------|-------|-----------|
| Horizon line height | bottom **15%** of viewport height (`y: 0.85` fraction) — within `05-CONTEXT.md`'s locked 12–18% range | Leaves ~85% of vertical space for star/Milky Way composition; the sky dominates, the ground reads as a thin near-black band |
| Ground fill | flat `--sky-zenith` fill (the darkest sky token — see Color) below the horizon line | "Pitch-dark ground" per `PROJECT.md`; the ground never needs its own token, it reads as the same darkest tone as deep sky, just below the horizon cut |
| Camper silhouette width | `clamp(160px, 16vw, 260px)` | Small enough to read as "a distant camp," never a hero object — the sky stays the hero |
| Camper silhouette anchor | bottom-left, `x: 0.06–0.22` of viewport width, base sitting on the horizon line | Chosen to stay clear of `DeckIndex`'s fixed bottom-right pill/jump-list (`04-UI-SPEC.md`) and to balance compositionally opposite the Milky Way's right-side core (see below) |
| Copper glow radius | 40–56px soft radial falloff around the window/ember point, peak alpha ≈0.55 | Mirrors Fig. 01's node-glow radial-gradient technique — layered radial fills only, never `shadowBlur`/CSS blur (`STACK.md`, `PITFALLS.md` Pitfall 7) |
| Firefly dot radius | 1.5–2.5px | Small enough to read as "sparks," never mistaken for stars |
| Firefly ground band | bottom 25% of viewport (`y: 0.75–1.0`) | Locked in `05-CONTEXT.md` Specifics |
| Firefly count | ≤15 (recommended starting density: **9**) | `05-CONTEXT.md` ceiling is locked; the exact count is Claude's discretion — 9 reads as "sparse enough to feel discovered," per the Specifics note |
| Firefly drift speed | 4–8px/s, random per-firefly, gentle wander (no straight lines) | "Drift slowly" per Specifics |
| Constellation star radius | 2.5–4px (ambient), always larger than the ambient field's largest band (2px) so a constellation always reads as visually distinct from the background field regardless of highlight state | Named-chapter stars must be findable even at rest |
| Constellation link stroke width | 0.75–1px hairline | "Thin, subtle" per `05-CONTEXT.md` — deliberately thinner than Fig. 01's 1px route stroke, since these are ambient/decorative, not a primary diagram |
| Firing beam head radius | 1.5px (Fig. 01 mirrors at 2.2px — scaled down ~⅓ since constellation links are shorter/quieter) | Mirrors the Fig. 01 beam MATH pattern per `05-CONTEXT.md`, reimplemented locally — not imported |
| Firing beam tail length | 40px (Fig. 01: 70px, scaled proportionally) | Same rationale |
| Firing beam speed | 0.18px/ms (Fig. 01: 0.22px/ms — slightly slower reads as "quieter") | Same rationale |
| Firing beam stroke width | 1.2px (slightly thicker than the link's own 0.75–1px hairline while traveling, so the pulse is perceptible against the static link) | — |
| Firing cadence | every 6–10s on a random link of the **ambient** (non-highlighted) sky, one at a time (`05-CONTEXT.md` locked range; exact random distribution is Claude's discretion — recommend uniform `6000 + random() * 4000` ms, same shape as Fig. 01's ambient-beam interval formula) | "Occasional," never simultaneous with a second firing |
| DPR cap | `min(devicePixelRatio ?? 1, 2)` — identical to Fig. 01 and the deck | Locked doctrine (`PROJECT.md` constraints, `STACK.md`) |

Exceptions beyond the above: none. This table **is** Phase 5's spacing/geometry exception set, matching the pattern `02-UI-SPEC.md` and `04-UI-SPEC.md` already established.

## Color

Zero hex literals outside `tokens.css` (locked doctrine, unchanged). This phase's ONLY token-file change: **4 new sky tokens appended to `tokens.css`**, per `05-CONTEXT.md`'s explicit allowance ("new sky tokens ALLOWED in tokens.css only").

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg: #0f1216` (unchanged) | Still the page's base — the scene renders as a full-bleed layer *underneath* `--bg`'s usual role (deck chrome, panel scrim base color), never replacing the token system, only extending it with deeper tones for the sky-specific canvas |
| Secondary (30%) | `--panel` / `--panel2` (unchanged) | Not used by the scene directly — panel content (text/cards) keeps its existing surfaces; the scene never paints a `--panel`-colored shape |
| Accent (reserved, <5% doctrine — unchanged) | `--accent: #d99163` | **Reserved for, this phase adds:** the camper window/ember glow (SKY-02) and the firefly pulse (SKY-03) — ONLY these two. Never used for constellation stars, links, or firing beams (see Sky Palette below) — this is a deliberate extension of the existing "never sprinkle accent everywhere" rule: the ONE warm note in the scene stays the camper + its sparse fireflies, exactly as `05-CONTEXT.md`'s Specifics state ("the camper's single copper-lit window/ember is the only warm note"). |
| Destructive | `--amber` (unchanged) | Unused this phase — no destructive actions exist in a passive scene |

### Sky Palette (NEW — 4 tokens, appended to `tokens.css` this phase only)

| Token | Recommended value | Usage | Status |
|-------|--------------------|-------|--------|
| `--sky-zenith` | `#05070a` | Top-of-sky / deepest tone; also the ground fill below the horizon line (silhouette band) | **Claude's discretion — recommended starting value, MUST be re-validated during the Milky Way spike** (`05-CONTEXT.md` Sequencing) before this phase's engine work begins |
| `--sky-horizon` | `#141a2c` | Bottom-of-sky-gradient tone, just above the horizon line — a touch lighter/bluer than zenith (a believable faint atmospheric/starlight-scatter glow near the horizon, still very dark) | Same — discretion, spike-validated |
| `--milkyway` | `#cfd9f2` | The Milky Way band's core tint — a pale, cool lavender-white, layered at low alpha (multiple passes, not one flat gradient — see Milky Way Composition below) so it reads as "the brightest surface on the page" without banding | Same — discretion, spike-validated; this is the single highest-risk visual value in the whole phase (per `STACK.md`'s LOW-confidence flag on the Milky Way compositing recipe) |
| `--star` | `#eef2fa` | The star field's base white (all magnitude bands are this ONE token at varying alpha — never a second white/off-white value) — deliberately a distinct token from `--ink` so star-brightness tuning never accidentally couples to UI text-color changes later | Same — discretion, spike-validated |

**Why these four, not more:** `05-CONTEXT.md` caps new tokens at "2-4 additions" — this is exactly 4, and each has a single, non-overlapping job (zenith = darkest, horizon = the gradient's other end + ground, milkyway = the one saturated/vivid tone, star = the one star-white). No fifth token is needed: fireflies and the camper glow both reuse `--accent`; constellation stars/links/beams all reuse `--star` (never `--milkyway`, which is reserved for the band itself, and never `--accent`, which is reserved for the camper/fireflies).

**Hue discipline (Pitfall 11 — "harmonized, not just adjacent"):** all four sit in the same blue-slate hue family `--bg`/`--hair`/`--hair2` already establish (blue channel consistently the largest component), just pushed darker (zenith) or bluer/cooler (horizon, milkyway, star) — this is a deliberate extension of the existing graphite system, not a second, clashing palette.

Accent reserved for (explicit, complete list — this phase's additions): camper window/ember glow (static SVG/CSS, Layer 1) · firefly pulse (canvas, Layer 2). Nothing else in the night-sky scene uses `--accent`. Constellation stars/links/firing-beams use `--star` exclusively.

## Scene Composition & Layering Contract

*(Supplementary — not template-required, but this is the most design-heavy phase of v2.0 and the executor needs unambiguous placement numbers, matching the depth `02-UI-SPEC.md`/`04-UI-SPEC.md` already established for their respective phases.)*

### Layer split (locked in `05-CONTEXT.md` — restated here for executor visibility)

- **Layer 0** — starfield + Milky Way, pre-rendered ONCE to an offscreen canvas per resize (chunked via `requestIdleCallback`, `setTimeout`-sliced fallback for browsers without it — defensive per `PITFALLS.md` Pitfall 6), blitted every frame via `drawImage`. Never recomputed per-frame.
- **Layer 1** — SVG silhouette + copper glow. Static DOM/CSS, never touches the rAF loop. The glow's "subtle flicker" (if implemented) is a CSS `@keyframes` opacity pulse on a pseudo-element/filter — **not** canvas, **not** rAF-driven (`STACK.md`'s explicit recommendation) — see Motion Contract below for the exact numbers.
- **Layer 2** — the ONLY per-frame canvas work: twinkle subset (~5–8% of ambient stars), ≤15 fireflies, and at most ONE constellation firing beam at a time.

### Star density & magnitude bands (Claude's discretion — true unknown pending the spike; recommended starting values)

Reference viewport: 1440×900 desktop. Scale count proportionally to viewport area on resize (floor ~350 stars on small mobile viewports, cap ~1200 on very large desktop viewports) — density should read consistent across sizes, not literally fixed-count.

| Band | Share of total | Star count (of 700 reference total) | Radius | Alpha (`--star` token) |
|------|------------------|----------------------------------------|--------|--------------------------|
| Faintest | ~65% | ~455 | 0.5–0.7px | 0.10–0.20 |
| Dim | ~22% | ~155 | 0.7–1.0px | 0.25–0.40 |
| Mid | ~10% | ~70 | 1.0–1.4px | 0.45–0.65 |
| Bright | ~3% | ~20 | 1.4–2.0px | 0.70–1.00 |

This is a power-law distribution (many faint, few bright), per `05-CONTEXT.md`'s explicit "realism beats uniform sprinkle" instruction — never generate stars with uniform size/alpha. The twinkle subset (~40 stars, per the Spacing table above) is drawn only from the Mid + Bright bands (the faint majority never twinkles — imperceptible and wasteful per-frame cost for zero visual payoff).

### Milky Way composition (HIGH-risk value — spike-validated before this phase's engine work proceeds)

- **Technique (default, zero-dependency):** layered scatter-dots + soft gradient passes using `--milkyway` at low alpha (recommend 3–5 stacked passes, each a soft elongated radial/linear gradient at alpha 0.04–0.10, offset slightly to avoid a single flat banded gradient) — per `STACK.md`'s default recommendation. Fallback (only if the spike finds this visibly banded/artificial): the ~1–2KB `simplex-noise` package, per `05-CONTEXT.md`'s approved-fallback clause (must pass the package legitimacy audit + exact pin before adoption).
- **Angle:** 20–25° from vertical.
- **Placement (the load-bearing composition rule — SKY-01 meets SKY-05 here):** the band's peak-brightness core (the top quartile of its opacity range) runs through the **right margin**, roughly `x: 0.80–0.98` at its brightest, tapering to near-zero brightness by `x: 0.65` on its inner (left) edge. The content column (per `04-UI-SPEC.md`/`deck.css`, `max-width: 880px` centered) occupies roughly `x: 0.19–0.81` on a 1440px-wide viewport — the Milky Way's bright core therefore sits entirely outside the column; only its faint inner taper may graze the column's outer ~10–15%, where the scrim (below) absorbs any residual contrast risk. This is the concrete implementation of `05-CONTEXT.md`'s "content sits over the DARKER sky regions by design."
- **Vertical extent:** runs the full sky height, entering near the top edge around `x: 0.55–0.65` and sweeping right-and-down to exit near the horizon around `x: 0.85–0.98` — angled toward the same side as the constellation clusters embedded near it (see below), away from the camper (bottom-left).

### Constellation placement (all 4 clusters live in the outer margins, never inside the content-safe column)

| Constellation | Region | Approx. bounds (`x`, `y` fractions) | Notes |
|---------------|--------|---------------------------------------|-------|
| `microsoft` | left margin, upper | `x: 0.06–0.20`, `y: 0.10–0.35` | Outside the column; clear of the camper |
| `samsung` | left margin, mid | `x: 0.06–0.20`, `y: 0.40–0.65` | Above the camper's silhouette top |
| `aws` | right margin, upper, embedded near the Milky Way's inner edge | `x: 0.80–0.95`, `y: 0.15–0.45` | Deliberately placed near the band's richest visual area — the most senior chapter sits in the sky's brightest neighborhood |
| `education-patents` | right margin, lower | `x: 0.80–0.95`, `y: 0.50–0.75` | Above the horizon, clear of `DeckIndex`'s bottom-right fixed pill |

Each cluster: 5–8 stars (recommend 6), connected by a small sparse graph (a path + 1–2 cross-links — recommend 4–6 total link segments per cluster) — **never** a fully-connected mesh, per the locked "never a plexus mesh" doctrine. No cluster's bounding box may extend into the content-safe column (`x: 0.20–0.80`) beyond a token few-pixel edge.

**Accessibility note (confirmed, not a gap):** constellations are decorative wayfinding *reinforcement* only. The authoritative panel-identity signal for assistive technology is Phase 4's already-shipped `aria-live` announcement (`04-UI-SPEC.md`/`deck.ts`'s `announce()`) — the canvas needs no duplicate accessible name.

### Scrim implementation (SKY-05)

- **Where it lives:** a `.panel::before` pseudo-element (new CSS rule in `deck.css` or a phase-5-specific stylesheet — planner's choice of file), `position: absolute; inset: 0; z-index: 0; pointer-events: none;`. `.panel > *` (already targeted by an existing `deck.css` rule) gains `position: relative; z-index: 1;` so panel content always paints above the scrim.
- **Gradient (vertical, using CSS Relative Color Syntax off `--bg` — avoids needing a fifth new token):**
  ```css
  .panel::before {
    background: linear-gradient(
      180deg,
      rgb(from var(--bg) r g b / 0) 0%,
      rgb(from var(--bg) r g b / 0.28) 18%,
      rgb(from var(--bg) r g b / 0.38) 45%,
      rgb(from var(--bg) r g b / 0.38) 70%,
      rgb(from var(--bg) r g b / 0.15) 92%,
      rgb(from var(--bg) r g b / 0) 100%
    );
  }
  ```
- **Ceiling:** peak opacity 0.38, inside `05-CONTEXT.md`'s ≤40% ceiling; the flat linear-gradient technique (no blur) matches `PITFALLS.md` Pitfall 7/8's explicit mitigation (bake contrast fixes into a cheap gradient, never a live blur filter).
- **Verification:** per `05-CONTEXT.md`, contrast must be checked at the WORST-CASE point — sample the brightest sky pixels that can appear under a text region (Milky Way taper edge, a Bright-band star, the constellation glow) via canvas readback or screenshot analysis, not an average. This is a Phase 5 quality-gate item, not a Phase 6 deferral.

## Motion & Reduced-Motion Contract

*(Supplementary — load-bearing for SKY-03/SKY-04/CONST-02/CONST-03, mirrors `02-UI-SPEC.md`'s Motion Contract structure.)*

### Ambient loop (normal motion)

| Element | Behavior |
|---------|----------|
| Twinkle | ~40 stars (Mid/Bright bands), independent sine-wave alpha wobble, period 2–4s (random phase + period per star), amplitude ±0.15–0.25 alpha around the star's base alpha — never a hard flicker |
| Firefly drift | 9 fireflies, gentle random-wander motion at 4–8px/s within the bottom-25% ground band, soft alpha pulse period 3–5s (alpha 0.4–0.9), copper-tinted (`--accent`) |
| Camper glow pulse | CSS `@keyframes` opacity pulse, period ~4–6s, opacity range 0.7–1.0 — Layer 1, CSS-only, never rAF/canvas |
| Constellation firing | one link at a time, ambient (non-highlighted) sky only, every 6–10s (`6000 + random() * 4000`ms), suppressed while paused or under reduced motion |
| Constellation brighten/dim | driven ONLY by `nightsky:panel-change` (Phase 4's shipped event, `04-01`/`04-02`) — never polled or DOM-inferred | 400ms ease-out (locked, `05-CONTEXT.md`) |

### Ambient/brightened/dimmed alpha states (constellation stars + links)

| State | Star alpha | Link alpha | Glow halo |
|-------|------------|------------|-----------|
| Ambient (resting, no panel mapped or this constellation's panel not active) | 0.40–0.55 | 0.12–0.20 | none |
| Brightened (this constellation's panel is the active panel) | 0.85–1.00 | 0.45–0.55 | radial halo, radius ≈1.5× star radius, alpha 0.20–0.30 |
| Dimmed (a sibling constellation is brightened) | 0.25–0.35 | 0.08–0.12 | none |

At most ONE constellation is ever brightened; when the active panel maps to none (`hero`/`contact`), all four sit at Ambient (not Dimmed).

### Pause / resume (SKY-04, Pattern 3 from `ARCHITECTURE.md`)

| Condition | Behavior |
|-----------|----------|
| `document.hidden` | rAF loop pauses natively; scene holds its last frame |
| Fig. 01 panel active (`nightsky:panel-change` detail `id === 'fig-01'`) | `scene.ts` calls its own `pauseAmbient()` — stops the rAF loop, paints one dim static frame; resumes via `resumeAmbient()` on leaving the panel. This is the "one-active-animation rule": the scene never co-runs its loop against Fig. 01's own 60fps loop |
| Transition lock (420ms, Phase 4) | No special handling needed — the scene's own loop is independent of the deck's transition lock, it simply keeps animating underneath a panel transition unless the incoming/outgoing panel is `fig-01` |

### Reduced motion (`prefers-reduced-motion: reduce`) — the ONE static-frame composition

Per `05-CONTEXT.md` and `PITFALLS.md` Pitfall 4 (stop, don't dampen):

| Layer | Reduced-motion behavior |
|-------|--------------------------|
| Layer 0 (starfield + Milky Way) | Unchanged — already static (drawn once regardless of motion preference) |
| Layer 1 (silhouette + glow) | Static SVG; glow pulse `@keyframes` is set to `animation: none` via `@media (prefers-reduced-motion: reduce)`, holding at a fixed mid-high opacity (recommend 0.85, the pulse's midpoint) rather than its lowest or highest extreme |
| Layer 2 — twinkle | Fully OFF — all stars render at their fixed base alpha (Mid/Bright bands render at their band's alpha ceiling, not wobbling) |
| Layer 2 — fireflies | Fully OFF — no fireflies rendered at all under reduced motion (this is "decorative," fully removable per WCAG C39, not merely dimmed) |
| Layer 2 — constellation firing | Fully OFF — never fires |
| Layer 2 — constellation brighten/dim | **Still occurs** (this is the one piece of "essential" wayfinding motion, per `PITFALLS.md` Pitfall 4's essential/decorative distinction) but the 400ms ease-out transition collapses to an **instant (0ms) state change** — the active panel's constellation is simply drawn already-brightened, no fade |
| Media query listener | `matchMedia('(prefers-reduced-motion: reduce)').addEventListener('change', …)` — re-branch live if the OS setting toggles mid-session, same doctrine Fig. 01 and the deck already established (`02-UI-SPEC.md`, `04-UI-SPEC.md`) |

Under reduced motion, **no rAF loop ever starts** for the night-sky scene — exactly Fig. 01's existing `renderStaticFrame` doctrine (`02-UI-SPEC.md`), applied to the new engine.
